## Search Endpoints

### Search Project

<CodeGroup>

```bash curl
curl -X POST http://localhost:3333/api/v1/search/project \
  -H "Content-Type: application/json" \
  -d '{
    "query": "authentication logic",
    "projectId": "my-project",
    "maxResults": 10,
    "minScore": 0.3
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3333/api/v1/search/project', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    query: 'authentication logic',
    projectId: 'my-project',
    maxResults: 10,
    minScore: 0.3
  })
});
const data = await response.json();
```

```python Python
import requests

response = requests.post(
    'http://localhost:3333/api/v1/search/project',
    json={
        'query': 'authentication logic',
        'projectId': 'my-project',
        'maxResults': 10,
        'minScore': 0.3
    }
)
data = response.json()
```

</CodeGroup>

**Endpoint:** `POST /api/v1/search/project`

**Description:** Contextual search using hybrid vector + keyword search with RRF ranking.

#### Request Body

<ParamField body="query" type="string" required>
  Search query (natural language or keywords)
</ParamField>

<ParamField body="projectId" type="string" required>
  Project ID to search in
</ParamField>

<ParamField body="projectPath" type="string">
  Project path for auto-reindex if needed
</ParamField>

<ParamField body="maxResults" type="number" default={10}>
  Maximum number of results to return
</ParamField>

<ParamField body="minScore" type="number" default={0.3}>
  Minimum relevance score (0-1)
</ParamField>

<ParamField body="responseMode" type="string" default="summary">
  Response format: `summary` or `full`
</ParamField>

<ParamField body="autoReindex" type="boolean" default={true}>
  Automatically reindex if project is stale
</ParamField>

<ParamField body="include" type="array">
  Glob patterns to include (e.g., `["*.ts", "*.tsx"]`)
</ParamField>

<ParamField body="exclude" type="array">
  Glob patterns to exclude (e.g., `["node_modules/**"]`)
</ParamField>

<ParamField body="explainScores" type="boolean" default={false}>
  Include score explanations in response
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates if the search was successful
</ResponseField>

<ResponseField name="data" type="object">
  Search results data
  
  <Expandable title="properties">
    <ResponseField name="results" type="array">
      Array of search results with file paths, content, and relevance scores
    </ResponseField>
    
    <ResponseField name="totalResults" type="number">
      Total number of results found
    </ResponseField>
    
    <ResponseField name="query" type="string">
      The original search query
    </ResponseField>
  </Expandable>
</ResponseField>

## Memory Endpoints

### Store Memory

**Endpoint:** `POST /api/v1/memory/store`

**Description:** Store a new memory in the hierarchical memory system (local SQLite).

<CodeGroup>

```bash curl
curl -X POST http://localhost:3333/api/v1/memory/store \
  -H "Content-Type: application/json" \
  -d '{
    "content": "User prefers TypeScript for new features",
    "type": "preference",
    "importance": 0.8,
    "tags": ["typescript", "preferences"]
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3333/api/v1/memory/store', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    content: 'User prefers TypeScript for new features',
    type: 'preference',
    importance: 0.8,
    tags: ['typescript', 'preferences']
  })
});
```

</CodeGroup>

#### Request Body

<ParamField body="content" type="string" required>
  Content to store in memory
</ParamField>

<ParamField body="type" type="string" required>
  Type of memory: `preference`, `conversation`, `code`, `decision`, or `pattern`
</ParamField>

<ParamField body="userId" type="string">
  User identifier for scoping memory
</ParamField>

<ParamField body="projectId" type="string">
  Project identifier for scoping memory
</ParamField>

<ParamField body="sessionId" type="string">
  Session identifier for scoping memory
</ParamField>

<ParamField body="agentId" type="string">
  Agent identifier for scoping memory
</ParamField>

<ParamField body="tags" type="array">
  Tags for categorizing the memory
</ParamField>

<ParamField body="importance" type="number" default={0.5}>
  Importance score (0-1)
</ParamField>

<ParamField body="format" type="string" default="toon">
  Output format: `json` or `toon`
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates if the memory was stored successfully
</ResponseField>

<ResponseField name="data" type="object">
  Stored memory details
  
  <Expandable title="properties">

      Unique identifier for the stored memory
    </ResponseField>
    
    <ResponseField name="content" type="string">
      The stored content

</ResponseField>

## Project Endpoints

### Index Project

**Endpoint:** `POST /api/v1/project/index`

**Description:** Start indexing a project directory in background. Returns a jobId immediately.

<CodeGroup>

```bash curl
curl -X POST http://localhost:3333/api/v1/project/index \
  -H "Content-Type: application/json" \
  -d '{
    "projectPath": "/home/user/my-project",
    "projectId": "my-project",
    "forceReindex": false,
    "warmCache": true
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3333/api/v1/project/index', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    projectPath: '/home/user/my-project',
    projectId: 'my-project',
    forceReindex: false,
    warmCache: true
  })
});
const { data } = await response.json();
console.log('Job ID:', data.jobId);
```

</CodeGroup>

#### Request Body

<ParamField body="projectPath" type="string" required>
  Absolute path to the project directory to index
</ParamField>

<ParamField body="projectId" type="string">
  Unique identifier for the project (defaults to directory name)
</ParamField>

<ParamField body="forceReindex" type="boolean" default={false}>
  Force reindexing even if project is already indexed
</ParamField>

<ParamField body="warmCache" type="boolean" default={false}>
  Pre-cache common queries after indexing completes
</ParamField>

<ParamField body="warmupQueries" type="array">
  Custom queries to pre-cache (e.g., `["authentication", "API routes"]`)
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates if the indexing job was started
</ResponseField>

<ResponseField name="data" type="object">
  Job information
  
  <Expandable title="properties">
    <ResponseField name="jobId" type="string">
      Unique identifier for tracking the indexing job
    </ResponseField>
    
    <ResponseField name="status" type="string">
      Initial job status (usually `pending` or `running`)
    </ResponseField>
  </Expandable>
</ResponseField>

## Context Endpoints

### Compress Context

**Endpoint:** `POST /api/v1/context/compress`

**Description:** Compress context using semantic compression (keeps structure, removes details).

<CodeGroup>

```bash curl
curl -X POST http://localhost:3333/api/v1/context/compress \
  -H "Content-Type: application/json" \
  -d '{
    "content": "...code or text content...",
    "strategy": "code_structure",
    "targetRatio": 0.7,
    "language": "typescript"
  }'
```

```javascript JavaScript
const response = await fetch('http://localhost:3333/api/v1/context/compress', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    content: codeContent,
    strategy: 'code_structure',
    targetRatio: 0.7,
    language: 'typescript'
  })
});
const { data } = await response.json();
console.log('Compressed:', data.compressed);
console.log('Ratio:', data.compressionRatio);
```

</CodeGroup>

#### Request Body

<ParamField body="content" type="string" required>
  Content to compress
</ParamField>

<ParamField body="strategy" type="string" default="code_structure">
  Compression strategy:
  - `code_structure` - Extract function signatures, types, exports
  - `conversation_summary` - Summarize conversation history
  - `semantic_dedup` - Remove semantically duplicate content
  - `hierarchical` - Multi-level compression
</ParamField>

<ParamField body="targetRatio" type="number" default={0.7}>
  Target compression ratio (0-1). Lower = more aggressive compression
</ParamField>

<ParamField body="language" type="string">
  Programming language for code compression (e.g., `typescript`, `python`, `javascript`)
</ParamField>

#### Response

<ResponseField name="success" type="boolean">
  Indicates if compression was successful
</ResponseField>

<ResponseField name="data" type="object">
  Compression results
  
  <Expandable title="properties">
    <ResponseField name="compressed" type="string">
      The compressed content
    </ResponseField>
    
    <ResponseField name="originalSize" type="number">
      Original content size in characters
    </ResponseField>
    
    <ResponseField name="compressedSize" type="number">
      Compressed content size in characters
    </ResponseField>
    
    <ResponseField name="compressionRatio" type="number">
      Achieved compression ratio (0-1)
    </ResponseField>
    
    <ResponseField name="tokenSavings" type="number">
      Estimated tokens saved
    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

All endpoints return errors in a consistent format:

```json
{
  "success": false,
  "error": "Descriptive error message"
}
```

### Common Errors

<AccordionGroup>
  <Accordion title="Project Not Indexed">
    **Error:** `Project 'xyz' not found. Please index it first.`
    
    **Solution:** Index the project using `POST /api/v1/project/index` before searching.
  </Accordion>

  <Accordion title="Ollama Not Available">
    **Error:** `Ollama service is not available`
    
    **Solution:** Ensure Ollama is running (`ollama serve`) and the embedding model is installed.
  </Accordion>

  <Accordion title="Invalid Parameters">
    **Error:** `Validation error: [field] is required`
    
    **Solution:** Check that all required parameters are provided in the request body.
  </Accordion>

  <Accordion title="Database Error">
    **Error:** `Database operation failed`
    
    **Solution:** Check that the data directory exists and has proper permissions.
  </Accordion>
</AccordionGroup>
